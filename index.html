Reviewing the Wallboard Displays

Once your desired filters are set, you can review the Wallboard displays.

Work Item Status

This chart shows the current state of all routed work items

  • Waiting: Case is in backlog, not yet assigned (no available agent/skill or all at capacity).
  • Assigned: Case has been routed and accepted by an agent but not yet being worked.
  • In Progress: Agent is actively working the case (opened or moved into a working state).
Agent Capacity

Shows two key values:

  • Available Capacity: How much capacity remains across online agents to take new work.
  • Used Capacity: How much capacity is currently being consumed by ongoing work.

This helps you instantly see whether your team has room to handle more or is overloaded.

Agent Capacity Status

Shows a breakdown of agents by their load:

  • At Capacity: Fully occupied—no more work can be assigned.
  • Available: Ready and able to take new work.
  • Busy: Logged in but marked as not accepting new items.
  • Idle: Online but currently handling no work.
  • Offline: Not logged in.

This bar or donut chart helps balance workload and identify who might be underutilized.

Agent Presence Statuses

Shows the distribution of agents by their actual presence:

  • Available, In Meeting, Focus Time, On Lunch, Etc.

Helps managers spot whether reps are spending too much time away, offline, or available but not receiving work.

Wait Time
  • This chart tracks how long unassigned work has been waiting in the backlog before an agent picks it up.
  • Often shown as average wait time or longest wait time.
  • Calculated from the moment work enters Omni-Channel until it’s assigned.
Agent Work Statuses

Tracks how many work items were:

  • Assigned: Work was routed and delivered to an agent.
  • Opened: Agent has actively opened/started working the item.
  • Closed: Work completed and resolved by the agent.
  • Unavailable: Work could not be routed because the agent wasn’t in an Available state.
  • Declined (does not apply): Agent explicitly rejected the routed work.
  • Push Timeout (does not apply): Agent didn’t respond in time, so the work offer expired.
  • Canceled (does not apply): Work was withdrawn (e.g., borrower ended chat, case pulled back).
Work Performance

Work Performance measures how effectively reps are handling the work once it’s assigned.

  • Average Work Handle Time: Total time from when work is assigned to when it’s completed/closed.
  • Average Speed to Answer (does not apply): How long work items wait before an agent accepts them.
  • Average Active Work Time (does not apply): The time agents spend actively engaged on the work (not just holding it).
  • After Conversation Time (does not apply): Wrap-up time agents spend finalizing notes or tasks after closing the work.
